Cross-Architecture Knowledge Distillation

CAKD — Luồng dữ liệu đi qua các block

Một ảnh vào cả hai model. Student ResNet-50 (trái) và Teacher ViT-B/16 (phải) chạy song song; 4 đầu ra mỗi bên căn theo hàng và hội tụ về các loss ở giữa, rồi cộng thành loss tổng. Chữ trong ngoặc = shape tensor, B = batch.

luồng Student luồng Teacher đầu vào phụ gộp loss

Cách đọc: mỗi hàng căn thẳng: đầu ra Student (trái) → loss (giữa) ← đầu ra Teacher (phải). Nhánh distill (attn, vit_feat) rẽ ra tại layer3; nhánh chính (logits, cls_token) đi tiếp qua layer4 → cnn_token. Đường đỏ bên phải gộp 4 loss thành loss tổng.

Epoch 0–25 · λ = 0 — chỉ cls_loss (tắt distill/GAN) Epoch 26–34 · λ: 0.02→0.18 — bật distill tăng dần Epoch 35–89 · λ = 0.2 — chạm trần, chạy hết

Khi chạy thật: chỉ giữ stem → layer1..4 → fc → logits; bỏ toàn bộ pca_proj / gl_proj / cls_proj, Discriminator và Teacher.

Chỗ bài báo sai / thiếu so với code

Bài báo: ℒtotal = (ℒproj1 + ℒproj2) + λ·ℒMVG (Eq. 4–10) — lệch code ở 7 điểm:

  1. Thiếu ℒcls và lịch λ(e). Code có CrossEntropy (thành phần chính) và λ thay đổi theo epoch (warmup từ epoch 25, trần 0.2); bài báo coi λ hằng số.
  2. Thiếu hệ số thật: code dùng 0.2 / 0.05 trong ℒpca, 1 / 1 / 0.05 trong ℒgl, và 0.05 trước số hạng adversarial-real.
  3. ℒgl rộng hơn ℒproj2. Bài báo chỉ khớp feature ‖hT−h′S‖². Code khớp 3 thứ: logits (MSE, không phải KL), class token, feature.
  4. Adversarial chạy trên ATTENTION MAP, không phải feature h. Bài báo cho D phân biệt hT vs h′S. Code cho D (PatchGAN) phân biệt attention teacher Ā(1)T vs student AqkS.
  5. GAN là vanilla (BCEWithLogits), không phải log(1−D) thuần. Số hạng generator là ℓ(D(AqkS),1) — bản non-saturating −log D, ổn định hơn.
  6. Số hạng adversarial-real trong loss student là "loss chết" (gradient = 0 do pred_real.detach()).
  7. Bất đối xứng softmax: student so điểm trước softmax (QKᵀ/√d) với xác suất sau softmax của teacher — khác thang đo. Có thể là lý do công thức attention bài báo trông "không khớp".
